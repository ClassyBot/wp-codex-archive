Description #Description

\n\t

The array structure is a little complex:

\n
$res = array(\n    'headers'  => array(),\n    'response' => array(\n        'code'    => int,\n        'message' => string\n    )\n);
\n

All of the headers in $res[‘headers’] are with the name as the key and the
\nvalue as the value. So to get the User-Agent, you would do the following.

\n
$user_agent = $res['headers']['user-agent'];
\n

The body is the raw response content and can be retrieved from $res[‘body’].

\n

This function is called first to make the request and there are other API functions to abstract out the above convoluted setup.

\n

Request method defaults for helper functions:

\n
    \n
  • Default ‘GET’ for wp_remote_get()
    • \n
  • Default ‘POST’ for wp_remote_post()
    • \n
  • Default ‘HEAD’ for wp_remote_head()
    • \n

    Sample ConvertKit API call function using wp_remote_request()

    \n
    \n
    \n
      private function call($args, $endpoint, $api_secret = null, $method = 'GET') {\r\n    if(is_null($api_secret)) { $api_secret = $this->api_secret(); }\r\n\r\n    //Populate the correct endpoint for the API request\r\n    $url                = "https://api.convertkit.com/v3/{$endpoint}?api_secret={$api_secret}";\r\n\r\n    //Allow 3rd parties to alter the $args\r\n    $args               = apply_filters('convertkit-call-args', $args, $endpoint, $method);\r\n\r\n    //Populate the args for use in the wp_remote_request call\r\n    $wp_args            = array('body' => $args);\r\n    $wp_args['method']  = $method;\r\n    $wp_args['timeout'] = 30;\r\n\r\n    //Make the call and store the response in $res\r\n    $res = wp_remote_request($url, $wp_args);\r\n\r\n    //Check for success\r\n    if(!is_wp_error($res) && ($res['response']['code'] == 200 || $res['response']['code'] == 201)) {\r\n      return $res['body'];\r\n    }\r\n    else {\r\n      return false;\r\n    }\r\n  }
