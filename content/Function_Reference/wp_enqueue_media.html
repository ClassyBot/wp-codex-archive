Description[edit]

Enqueues all scripts, styles, settings, and templates necessary to use all media JavaScript APIs.

This function should be called from the 'admin_enqueue_scripts' action hook, or later.

Usage[edit]

<?php wp_enqueue_media( $args ); ?>

Parameters[edit]

$args
(array) (optional) An array of arguments. Currently only one argument is used, 'post', which if passed will be used to set up the scripts for that post. Note: if you do not set this parameter, uploaded files will not be attached to a post.
Default: array()

Returns Values[edit]

This function does not return a value.

Examples[edit]

Default usage: 

wp_enqueue_media();

Enqueue the media scripts for a particular post: 

<?php

$args = array( 'post' => 34 );

wp_enqueue_media( $args );

?>

Notes[edit]

  • Uses: do_action() - calls 'wp_enqueue_media' after enqueing the media.
  • Uses: did_action( 'wp_enqueue_media' ) - Makes sure that the function is run only once.
  • Uses: apply_filters():
    • Calls 'media_upload_tabs' with an array of the old thickbox media tabs to ensure plugins will work. The default tabs will then be reset.
    • Calls 'disable_captions' to check whether captions should be disabled.
    • Calls 'media_view_settings' with the array of settings and the post object.
    • Calls 'media_view_strings' with the array of strings used in the media viewer and the post object.
  • Uses: wp.media (JavaScript) to call the media uploader
    • Documentation: http://www.grafikart.fr/tutoriels/wordpress/wp-media-uploader-403 (French video)

Change Log[edit]

Since: 3.5.0
