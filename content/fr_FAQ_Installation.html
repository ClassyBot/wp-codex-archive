Dois-je Créer une Base de Données ?[edit]

WordPress requiert l'accès à une base de données MySQL (en anglais) pour stocker les informations. Vous avez donc besoin d'une base de données.

Vous pouvez créer une nouvelle base de données si :

  1. Vous n'en avez pas déjà créé une sur le serveur.
  2. Votre généreux hébergeur vous offre plus qu'une base de données, et vous souhaitez avoir une base séparée pour le blog que vous allez configurer.

Il n'est pas essentiel de créer une nouvelle base de données pour chaque installation de WordPress.

  • Si vous utilisez la même base de données pour plusieurs installations de WordPress, veillez à éditer le fichier wp-config.php pour vous assurer que chaque installation à bien un préfixe unique de table de base de données.
  • Si vous configurez une nouvelle base de données pour un nouveau blog, éditez wp-config.php, et assurez-vous d'avoir les bonnes informations concernant la base de données.

Consultez également les Instructions Détaillées de l'Installation.

Qu'est-ce que l'Erreur 403 que j'Obtiens quand j'Essaie de Voir mon Site ?[edit]

Symptômes : Vous ne semblez pas pouvoir vous connecter à votre compte administrateur après avoir utilisé les bons nom d'utilisateur et mot de passe. Vous obtenez un message de ce type : 

HTTP Error 403 - Forbidden

Solution : Si votre compte est hébergé sur un serveur Windows, le problème peut venir des Indices de Répertoire (Directory Indexes). La configuration par défaut du serveur est de charger les fichiers index.htm, index.html, default.htm, default.html, default.asp.

WordPress est conçu en PHP donc la page par défaut est index.php. Cela ne fait pas partie de la configuration par défaut et nous devons donc l'ajouter. Pour faire ça, vous devons cliquer sur Options Web (Web Options) dans le Panneau de Contrôle (Control Panel). Puis faites défiler la page jusqu'à la section Indices de Répertoire (Directory Indexes) pour ajouter index.php aux Indices de Répertoire (Directory Indexes).

Si nécessaire, contactez votre hébergeur pour plus d'aide dans ce domaine.

Puis-je Renommer le Répertoire WordPress ?[edit]

Si vous n'avez pas encore installé WordPress, vous pouvez renommer le répertoire contenant les fichiers WordPress, avant ou même après les avoir téléchargé.

Si vous avez déjà installé WordPress, et que vous souhaitez renommer le répertoire, connectez-vous en tant qu'administrateur et changez les paramètres suivants dans Options > Général :

  • Adresse de WordPress (WordPress address) (URI)
  • Adresse du blog (Blog address) (URI)

Une fois cette étape effectuée, vous pouvez renommer le répertoire contenant les fichiers WordPress.

Consultez également l'article Déplacer WordPress (en anglais).

Pourquoi mon Répertoire de Téléchargement est-il "C:apachehtdocswordpress" ?[edit]

Puis-je Installer WordPress sur Windows 2000 ?[edit]

N'UTILISEZ PAS une base de données MySQL (en anglais) en version 4.1.7 si vous essayez d'installer WordPress sur une plate-forme Windows. Veuillez lire cet article écrit à l'orgine sur un des forums :

(Texte original en anglais)
"A tous ceux qui rencontrent des problèmes en installant WordPress sur votre propre station de travail Windows 2000 (et les autres versions du système d'exploitation) - et peut-être à certains hébergeurs également: N'utilisez pas MySQL 4.1.7 - le problème vient de là si vous obtenez le message d'erreur "Impossible de se connecter à la base de données ! (Error establishing a database connection !)" Il semblerait qu'il y ait des incompatibilités avec les autres composants. Utilisez MySQL 4.0.22 à la place.
Merci à la ADAMANT pour sa réponse à l'article d'ANTOINE le 6 novembre 2004 à 02h21, l'installation de WordPress ne prit effectivement que 5 minutes, après deux jours de frustration :
J'ai commencé avec Windows 2000, PHP 4.3.9, Apache 1.3.33 et MySQL 4.1.7. J'ai passé deux jours à littéralement vérifier 100 fois mon wp-config.php ; faisant des modifications ; cherchant des erreurs dans ma base de données, utilisant tous les noms connus pour mon hôte (localhost, 127.0.0.1, <adresse IP>, nom de l'ordinateur, etc...) jusqu'à ne plus avoir de solution. J'obtenais toujours :
"Impossible de se connecter à la base de données ! (Error establishing a database connection !)"
Je savais qu'il ne pouvait pas venir de ma configuration. Donc merci pour la suggestion d'ADAMANT, j'ai désintallé MySQL 4.1.7 et téléchargé puis installé MySQL 4.0.22 depuis mysql.org. Après avoir installé et configuré les nouveau (ancien) MySQL, ce qui prit approximativement 7 minutes (très simple), j'ai lancé le script install.php une fois de plus et OUI !! Il fallut effectivement moins de 5 minutes pour installer WordPress.
NOTE : le site web d'Apache conseille de NE PAS utiliser Apache 2 en production. Donc, après tout ce temps perdu avec le dernier MySQL, je suggère la configuration suivante à ceux qui souhaitent contrôler leur environnement :
PHP 4.3.9, Apache 1.3.33 et MySQL 4.0.22. Sur Windows tout du moins, ils fonctionnent très bien ensembles."
Steven

MISE A JOUR : MySQL 4.1.7 utilise un nouvel algorithme de cryptage pour les mots de passe qui est incompatible avec les anciennes méthodes. Si vous voulez faire marcher WordPress avec une base de données 4.1.7, vous devez vous assurer que votre mot de passe est défini selon l'ancienne méthode. -- Nabil

ADDITIONAL : Comme précisé ci-dessus par Nabil, vous pouvez obliger les version ultérieurs à 4.1.7 de MySQL d'utiliser l'ancienne longueur de mot de passe. Depuis l'interface en ligne de commande de MySQL, en tant qu'utilisateur abilité à modifier les tables MySQL (généralement root) : 

USE mysql;
SET PASSWORD FOR 'utilisateurwordpress'@'localhost' = OLD_PASSWORD('unmotdepasse');
FLUSH PRIVILEGES;

Comme les méthodes normales concernant les mots de passe seront d'utiliser le nouveau style de cryptage, une autre solution serait de faire fonctionner le service MySQL avec le paramètre --old-password, mais la méthode ci-dessus fonctionne pour les versions 4.1.19 et 4.1.20 sur une installation XP. - Grey

Note : Les commentaires ci-dessus ont été ajoutés par des utilisateurs, et peuvent être subjectif dans leur contenu. Nous espérons qu'ils auront été d'une certaine aide pour les utilisateurs confrontés au problème spécifique abordé par cette question.

Comment Cacher mon blog au Public ?[edit]

Que vous soyez en train de tester une nouvelle version de WordPress, configurer un nouveau blog ou pour tout autre raison que vous ayez de limiter l'accès, les informations qui suivent peuvent vous aider à garder les visiteurs indésirables hors de portée.

Apache

Il n'y aucune garantie avec cette méthode. Vous pouvez utiliser le fichier .htaccess (qui contient également votre programmation des permaliens) pour vérifier les adresses IP et en empêcher certaines d'accéder à votre site. Cela n'arrêtera que les adresses IP, pas les personnes, et donc si elles ont accès à des adresses IP autorisées, elles pourront atteindre votre page. Un tutorial est disponible chez Clockwatchers.com (en anglais).

Un fichier .htaccess peut aussi être utilisé pour éviter les liens directs vers vos images (et le vol de bande-passante) ou pour configurer un blog protégé par mot de passe.

Authentification Basique d'Apache

Pour obliger la saisie d'un mot de passe à l'accès de votre site en utilisant les fichiers .htaccess et .htpasswd : Clockwatchers.com .htpasswd (en anglais).

Des outils pour vous aider dans la création des fichiers nécessaires à la protection par mot de passe de votre site : Clockwatchers.com Outils .htaccess et .htpasswd (en anglais).

Note : Lors d'un accès à votre site, le mot de passe est crypté basiquement en Base64 et peut facilement être intercepté et décodé.

Authentification Basique d'IIS pour Windows

Pour obliger la saisie d'un mot de passe à l'accès de votre site s'il est hébergé avec IIS, vous pouvez déselectionner Autoriser l'Accès Anonyme (Allow Anonymous Access) et sélectionner Authentification Basique (Basic Authentication). Vous aurez également besoin d'un nom d'utilisateur et d'un mot de passe.

Note : Lors d'un accès à votre site, le mot de passe est crypté basiquement en Base64 et peut facilement être intercepté et décodé.

MySQL[edit]

Ai-je Vraiment Besoin de MySQL ?[edit]

Il est obligatoire de disposer de MySQL (en anglais) (version 3.23.23 ou ultérieure) pour faire fonctionner votre blog WordPress. MySQL est une base de données relationnelle, et WordPress fonctionne uniquement avec MySQL.

Puis-je Utiliser une Autre Base de Données que MySQL ?[edit]

Les autres bases de données ne sont pas supportées pour le moment.

Il existe de nombreux autres excellents gestionnaires de base de données, tels que PostgreSQL ou SQLite auquel WordPress s'intéresse pour les supporter dans le futur. Prendre en charge plusieurs bases de données est plus compliqué qu'il n'y paraît et ne fait pas partie des développements actuels, bien que de nombreuses discussions aient été engagées sur la meilleure approche à envisager. Ces approches pour augmenter le nombre de bases de données prises en charge sont étudiées dans l'article Utiliser des Bases de Données Alternatives (en anglais). Il existe un portage de WordPress pour PostgreSQL appelé WordPress-Pg (en anglais).

Pourquoi WordPress Utilise-t-il MySQL ?[edit]

MySQL (en anglais) est extrêmement rapide. C'est également le serveur de base de données le plus répandu dans le monde. Gratuit et open-source, MySQL (en anglais) est pris en charge par des milleurs d'hébegeurs Linux (en anglais) (et Windows !) à bas prix, ce qui constitue un tout petit obstacle pour qui que ce soit désirant débuter un site WordPress (ou contrôlé par une base de données). La documentation de MySQL (en anglais) est utile, convaincante et minutieuse (Note : cela paraîtra peut-être intimidant si vous êtes novice dans le domaine). Ajouté à ça le fait que les utilisateurs peuvent manipuler directement MySQL (en anglais) avec phpMyAdmin (en anglais), développement spécifiquement dans ce but, et il apparaîtra évident que MySQL (en anglais) est le meilleur choix. Bien entendu, WordPress ne choisi que le meilleur.

Qu'est-ce que phpMyAdmin ?[edit]

phpMyAdmin est un outil écrit en PHP destiné à administrer MySQL via le web.

Ai-je Besoin de Connaître PHP pour Utiliser WordPress ?[edit]

Non. Le seul moment où vous pourriez modifier votre blog WordPress avec PHP (en anglais) serait lors de l'intégration de certains Plugins (en anglais). Dans la plupart des cas, des instructions claires sont fournies dans un fichier texte qui accompagne le plugin (en anglais). En dehors de ça, vous ne devriez pas changer de fichier PHP.

WordPress Fonctionne-t-il avec le Safe Mode de PHP ?[edit]

Tout à fait ! Il n'y a pas de problème connu avec une quelconque version de WordPress quand PHP (en anglais) est éxécuté en Safe Mode.

Ai-je Vraiment Besoin de PHP ?[edit]

Vous avez absolument besoin de PHP (en anglais) (version 4.1 ou ultérieure) pour faire fonctionner votre blog WordPress. PHP (en anglais) est le langage de programmation qui fait fonctionner l'ensemble de WordPress, et sans PHP (en anglais), votre serveur ne sera pas capable d'interprêter les pages qui créent votre blog.

Comment Savoir de Quelle Version de PHP je Dispose ?[edit]

Pour obtenir des informations concernant votre serveur, vous pouvez utiliser la fonction phpinfo(). Elle vous donnera des informations sur vos versions d'Apache (en anglais) et de PHP (en anglais) et mod_rewrite (en anglais).

Mise à Jour[edit]

Quelle Version de WordPress ai-je ?[edit]

Il existe plusieurs méthodes pour retrouver la version de WordPress que vous utilisez :

  • Dans le répertoire wp-includes de votre installation WordPress, ouvrez le fichier version.php avec un éditeur de texte (en anglais). Vous pourrez y retrouver le numéro de version de votre installation.
  • Dans le panneau d'administration de WordPress, le numéro de version est visible au bas de toutes les pages.
  • Certains thèmes incluent le numéro de version dans leur modèle de pied de page. Ou vous pouvez l'ajouter vous-même :
<?php bloginfo('version'); ?>

Après Avoir Exécuté upgrade.php, j'Obtiens le Message "Il semblerait que vous n'avez pas installé WordPress pour le moment. Essayez d'éxécuter install.php." ("It doesn't look like you've installed WP yet. Try running install.php")[edit]

Après avoir effectué la Mise à Jour de WordPress, et cliqué sur le lien "Bon bloggin (Have Fun", vous obtenez le message "Il semblerait que vous n'avez pas encore installé WP. Essayez d'éxécuter install.php. (It doesn't look like you've installed WP yet. Try running install.php.)". Le problème peut venir du fait qu'il n'existe pas d'utilisateur disposant des droits administratifs (en anglais).

Pour corriger ce problème, utilisez phpMyAdmin (en anglais) pour donner manuellemment ces droits à un utilisateur :

  1. Retrouvez tout d'abord l'ID de l'utilisateur que vous voulez rendre administrateur en consultant la table wp_user (en anglais).
  2. Dans la table wp_usermeta (en anglais), pour cet ID, trouvez le user_id correspondant, et là où meta_key égale wp_capabilities, changez meta_value en a:1:{s:13:"administrator";b:1;}.
  3. Dans la même table, pour ce user_id, là où meta_key égale wp_user_level, changez meta_value en 10.

Veuillez noter que le préfixe de ces tables et des deux meta_key utilise le préfixe de base de données que vous avez saisi dans le fichier wp-config.php. Donc, si votre préfixe de base de données est mywp_, les tables seront mywp_user et mywp_usermeta et les deux meta_key seront mywp_capabilities et mywp_user_level.

Consultez également le sujet de Mars 2006 du Forum de Support WordPress (en anglais).
