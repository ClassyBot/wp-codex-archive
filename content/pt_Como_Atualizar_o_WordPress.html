Atualização Automática[edit]

As versões recentes do WordPress possuem um recurso de atualização automática. Você pode iniciar a atualização clicando no link que aparece no sistema WordPress "Está disponível a nova versão XX" (se estiver lá) ou indo ao menu Ferramentas -> Upgrade.

Às vezes a Atualização automática pode falhar, por isso lembre-se de fazer backup de seus dados e desativar todos os plugins antes de iniciar a atualização.

Todos os seus arquivos precisam ser de propriedade do usuário administrador, sob o qual o servidor Apache executa, ou você receberá uma caixa de diálogo solicitando "Informações de conexão", e não importa o que informar, não vai funcionar.

Se a atualização automática não funciona para você, não entre em pânico, tente uma atualização manual.

Atualização Manual em Três Passos[edit]

Estas são as instruções curtas, se quiser mais, confira as instruções de Atualização Extendida. Se tiver problemas com o Passo Três da atualização, você pode querer ver as instruções da Atualização Extendida.

Para estas instruções, presume-se que o URL do seu blog é http://example.com/wordpress/. Durante o processo de atualização, o seu blog não vai estar disponível para seus visitantes.

Um aviso antes de começar[edit]

Se encontrar problemas para atualizar o WordPress com os Três Passos descritos aqui, é necessário reverter para a versão antiga antes de usar as instruções da Atualização Extendida, (ou seja, restaurar o backup feito no passo 0). Mesmo que você não corre qualquer risco neste processo, você pode encontrar alguns problemas no futuro. Então, pode não ser possível reverter o bastante para a versão anterior para corrigir o problema sem perder as alterações recentes.

Então, se você usar plugins e temas que não sejam os que vêm com o WordPress por padrão, é aconselhável começar com as instruções de Atualização Extendida.

Passo 0: Antes de Começar[edit]

  • Apenas no caso de algo dar errado, faça um backup de seus dados. Você pode saber mais sobre isso em Backups no WordPress
  • Certifique-se que o nome de usuário do banco de dados registrados para WordPress tem permissão para criar, modificar e excluir tabelas do banco de dados. Se você instalou o WordPress da forma padrão, e nada mudou desde então, está tudo como deve ser.
  • Desative todos os plugins. Um dos plugins pode não ser compatível com a nova versão, por isso é bom buscar por novas versões dos mesmos ou desativar qualquer que pode causar problemas. Você pode reativar plugins um por um após a atualização. Isto é particularmente importante quando a atualização para o WordPress 2.7!

Passo 1: Substituir o WordPress[edit]

1. Faça download da última versão do WordPress. Faça o download e extraia para seu computador ou baixe diretamente para o servidor. 1. Como um lembrete, para extrair um tar.gz para uma pasta usar este comando, substituindo (nome da pasta) com o nome de sua pasta:

folder: tar -xvzf latest.tar.gz -C ./(nome da pasta)

2. Delete as pastas wp-includes e wp-admin. O mais fácil é renomeá-las para o caso de algo dar errado, você ainda poderá reverter a situação.

3. Copie os arquivos novos do WordPress para o servidor, sobrescrevendo os arquivos antigos, exceto talvez a pasta wp-content (onde estão seus temas e plugins). Você pode usar FTP ou comandos shell para fazer isso ou usar o serviço do servidor. Veja só, voc~e deve substituir todos os arquivos do diretório raiz do WordPress!

NOTA: A pasta wp-content requer tratamento especial, assim como os plugins e pastas de temas. Faça download da pasta para seu computador e re-envie depois que tudo estiver funcionando.

Tome cuidado para preservar o conteúdo do arquivo wp-config.php no diretório raiz! Este arquivo contém as configurações atuais para a sua instalação existente, por exemplo, formato de dados. Ocasionalmente, as novas versões do WordPress adicionam instruções para este arquivo. (Por exemplo, na versão 2.5 a variável SECRET_KEY foi adicionado). Compare o arquivo existente com o novo arquivo de instalação que é chamado wp-config-sample.php. Você pode copiar o antigo arquivo wp-config.php, renomeá-lo para para BKwp-config.php, crie um novo arquivo wp-config.php e adicione as novas chaves se houver.

Passo 2: Atualizar a sua Instalação[edit]

Visite sua página index.php em WordPress/wp-admin. Você pode ser convidado a se logar novamente. Se uma atualização de banco de dados é necessária, neste ponto, WordPress irá detectá-la e dar-lhe um link para uma URL como http://example.com/wordpress/wp-admin/upgrade.php. Siga esse link e siga as instruções. Isto irá atualizar o banco de dados para ser compatível com o código mais recente. Se você não fizer este passo, seu blog pode ficar estranho...

Passo 3: Faça algo bom para si mesmo[edit]

Se você tiver o cache ativado, suas alterações serão exibidas para os usuários imediatamente se você limpar o cache neste ponto.

A instalação do WordPress foi atualizado com êxito!

Considere-se recompensando com um blog atualizado! Pode respirar aliviado!

Problemas[edit]

Se alguma coisa deu errado, a primeira coisa a fazer é passar por todas as etapas de nossas instruções de Atualização Extendida. Essa página também tem informações sobre alguns dos problemas mais comuns que podem ocorrer.
