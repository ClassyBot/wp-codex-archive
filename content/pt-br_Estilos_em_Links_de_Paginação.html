Você sabia que pode quebrar uma postagem ou página muito longa em várias partes, apenas inserindo <!--nextpage--> no texto? Apenas clique onde quer quebrar o texto e adicione <!--nextpage-->. Assim, uma postagem ou página muito longa é quebrada em várias páginas e o visitante terá links para ir de uma página a outra da mesma postagem e página. Algo como:

Páginas 1 2 3
Artigo ou Seção com explicações avançadas
A seguir, estão explicações ou instruções para usuários avançados, o que pode exigir o conhecimento de termos e ferramentas não comuns a todas as pessoas.
Adicione {{Avançado}} para usar esta caixa.

Sobre[edit]

O visual padrão para os links de páginas é mostrado acima. A página web está em destaque não é como um link, mas apresentado como um número sólido. Se você faz apenas algumas de várias páginas posts, isso pode ser tudo que você precisa para o leitor continuar a ler as páginas em seqüência.

Há duas tags de modelos que podem ser usadas ​​para links de páginas. A link_pages() e wp_link_pages() template tags. A tag link_pages() usas strings em aspas como parâmetros e wp_link_pages() usa bool como parâmetros O uso padrão, mostrado acima, para a tag wp_link_pages()é: 

<?php wp_link_pages(); ?>

Os parâmetros para usar com esta tag de modelo são:

  • before: Texto para colocar antes de todos os links. O padrão é <p>Pages:.
  • after: Texto para colocar depois de todos os links. O padrão é</p>.
  • next_or_number: Indica se números de página devem ser usados. Os valores válidos são number (Default) e next
  • nextpagelink: Texto para o link para a página seguinte. O padrão é Next page.
  • previouspagelink: (string) Texto para o link para a página anterior. O padrão é Previous page.
  • pagelink: String de formato para número de páginas. O  % na string do parâmetro é substituído pelo número da página, então Page % gera "Page 1", "Page 2", etc. O padrão é %, ou seja, apenas o número da página.

Você já viu a aparência padrão. Vamos brincar com algumas outras possibilidades.

Adicionando Estilo[edit]

Lembre-se você deve adicionar entidade de caracteres para letras acentuadas, tanto em arquivos de temas como de plugins. Veja uma lista de caracteres no artigo HTML ISO-8859-1.

Por padrão, aos links da página estão em uma tag de parágrafo HTML. Adicione uma classe CSS para a seção DIV em torno da tag de parágrafo e você tem ainda mais controle sobre a aparência dos links de páginas.

No seguinte exemplo: adicionamos uma tag DIV chamada pagelink, antes e após o código de links de página, mostramos a palavra "Página" ao lado de cada número e quando você estiver no fim da página, verá o número da página em vez da palavra "next", além de listar as páginas como mostrado abaixo.

Página 1 Página 2 Página 3
<?php wp_link_pages('pagelink=Página %'); ?>

Um estilo mais comunicativo:

Para ler mais, você pode voltar para a página anterior ou ler a página seguinte.
<?php wp_link_pages('before=To read this story, &after=. &next_or_number=next
&previouspagelink=você pode voltar para a página anterior
&nextpagelink= ou ler a página seguinte'); ?>

O código CSS para a classe pagelink mostrado nos dois exemplos: 

.pagelink { font-weight:bold, color:green}
.pagelink a{ font-weight:bold, color:green; text-decoration:underline}
