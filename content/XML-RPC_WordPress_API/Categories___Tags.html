wp.getCategories[edit]

Retrieve list of categories.

Parameters[edit]

  • int blogid
  • string username
  • string password

Return Values[edit]

  • array
    • struct
      • string categoryId
      • string parentId
      • string categoryName
      • string categoryDescription
      • string description: Name of the category, equivalent to categoryName.
      • string htmlUrl
      • string rssUrl

Errors[edit]

  • 401
    • If the user does not have the edit_posts cap.

Replacement[edit]

As of WordPress 3.4, it is preferable to use the wp.getTerms method, with category as the taxonomy_name parameter.

wp.suggestCategories[edit]

Retrieve list of categories that start with a given string.

Added in WordPress 2.2.

Parameters[edit]

  • int blogid
  • string username
  • string password
  • string category: Beginning of category name.
  • int max_results

Return Values[edit]

  • array
    • struct
      • string category_id
      • string category_name

Errors[edit]

  • 401
    • If the user does not have the edit_posts cap.

Replacement[edit]

As of WordPress 3.4, it is preferable to use the wp.getTerms method, with category as the taxonomy_name parameter and the category prefix as the search parameter.

wp.newCategory[edit]

Create a category.

Added in WordPress 2.2.

Parameters[edit]

  • int blogid
  • string username
  • string password
  • struct category
    • string name
    • string description
    • int parent_id
    • string slug

Return Values[edit]

  • int category_id

Errors[edit]

  • 403
    • If the user does not have the manage_categories cap.

Replacement[edit]

As of WordPress 3.4, it is preferable to use the wp.newTerm method, with category as the taxonomy_name parameter.

wp.deleteCategory[edit]

Delete an existing category.

Added in WordPress 2.5.

Parameters[edit]

  • int blogid
  • string username
  • string password
  • int category_id

Return Values[edit]

  • bool true

Errors[edit]

  • 401
    • If the user does not have the manage_categories cap.

Replacement[edit]

As of WordPress 3.4, it is preferable to use the wp.deleteTerm method, with category as the taxonomy_name parameter.

wp.getTags[edit]

Retrieve list of all tags.

Added in WordPress 2.7.

Parameters[edit]

  • int blogid
  • string username
  • string password

Return Values[edit]

  • array
    • struct
      • int tag_id
      • string name
      • string slug
      • int count
      • string html_url
      • string rss_url

Errors[edit]

  • 401
    • If the user does not have the edit_posts cap.

Replacement[edit]

As of WordPress 3.4, it is preferable to use the wp.getTerms method, with post_tag as the taxonomy_name parameter.
