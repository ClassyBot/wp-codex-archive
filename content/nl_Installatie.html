De beroemde 5 minuten-installatie[edit]

Voor de meesten is de installatie van WordPress heel gemakkelijk en kost het slechts vijf minuten voordat je kunt beginnen bloggen. Voordat je begint, zorg ervoor dat je host aan de WordPress server requirements voldoet.

  1. Unzip het bestand dat je juist gedownload hebt.
  2. Maak een database aan en ken een gebruiker toe. Dit doe je via het control panel waarvan je de gegevens van je host gekregen hebt.
  3. Open wp-config-sample.php met het kladblok en vul je database gegevens in.
  4. Bewaar en hernoem het bestand naar wp-config.php. Wanneer je moeilijkheden ondervindt met wp-config.php of als er iets is dat je niet helemaal begrijpt, bezoek dan de Editing wp-config.php pagina.
  5. Upload alle bestanden.
  6. Voer het installatiebestand uit door het te openen in je favoriete browser en naar install.php te navigeren. Afhankelijk van waar je de bestanden hebt gezet, kun je het vinden in de wp-admin folder: wp-admin/install.php.
    Voorbeeld: http://www.example.com/wp-admin/install.php

Dat is alles.

Aan het eind van de installatieprocedure wordt je een login usernaam en een willekeurig wachtwoord gegeven. Schrijf het wachtwoord op. Wanneer je dit wachtwoord vergeet, kun je niet inloggen op je blog. Wanneer je eenmaal ingelogd bent, kun je het wachtwoord veranderen in wat je maar wilt op de Gebruikers > Je Profiel pagina.

Wanneer je klaar bent, kun je je nieuwe door WordPress aangedreven site aankondigen in het "Your WordPress" forum.

Voorkomende Installatie Problemen[edit]

Vraag: Ik zie een hoop onzin op mijn pagina. Wanneer ik naar de broncode kijk, zie ik een heleboel vreemde zaken zoals: 

<?php 
require('./wp-blog-header.php');

Antwoord: Wanneer je dergelijke code op je pagina ziet, betekent dit dat de server de PHP code waarop WordPress stoelt, niet goed weergeeft. Check of je host aan de vereisten voldoet om WordPress te draaien, en contacteer de host voor assistentie.


Vraag: Tijdens de installatie krijg ik een foutboodschap die zegt: 

Fout bij het maken van de database-connectie

Antwoord: Check het volgende:

  • Ben je er zeker van dat je de juiste combinatie van gebruikersnaam/wachtwoord hebt gebruikt? Dit kan verschillen van de combinatie die je gebruikt voor e-mail of FTP.
  • Ben je er zeker van dat je de juiste hostnaam hebt ingetypt? Sommige hosts vereisen dat je localhost gebruikt, anderen hebben een specifieke hostnaam die kan verschillen van je web hostnaam, zoals dbserver.host.com.
  • Ben je er zeker van dat de database server draait?

Check (en dubbelcheck) de parameters in je wp-config.php bestand. Aangenomen dat de database die je gebruikt, ook daadwerkelijk bestaat, betekent dit altijd dat je iets verkeerd hebt ingevuld. Hier is een voorbeeld van hoe het relevante deel eruit zou kunnen zien: 

define('DB_NAME', 'john_db');      // De naam van de database die je gebruikt
define('DB_USER', 'john');         // Je MySQL database gebruikersnaam
define('DB_PASSWORD', 'p@$$w)rd'); // Je MySQL wachtwoord
define('DB_HOST', 'localhost');    // of dbserver.host.com - geen http://
