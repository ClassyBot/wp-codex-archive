30 ноября 2010 года вышел WordPress 3.0.2. Это обязательное обновление для всех предыдущих версий WordPress.

Версия базы данных в 3.0.2 (параметр db_version в wp_options) осталась прежней — 15477.

Установка/Обновление[edit]

Чтобы скачать WordPress 3.0.2, воспользуйтесь автоматическим обновлением в меню «Консоль» > «Обновления», либо перейдите на страницу http://ru.wordpress.org/releases/.

Пошаговые инструкции по установке и обновлению WordPress:

  • Обновление WordPress

Если вы новичок в WordPress, можно начать со следующих статей:

  • Новичкам в WordPress — С чего начать
  • Первые шаги с WordPress или Обновление WordPress (подробно)
  • Уроки WordPress

Содержание[edit]

  • Исправлена ошибка, позволявшая злоумышленнику с правами автора получить более полный доступ к сайту. (r16625)

Другие исправления и изменения для улучшения безопасности:

  • Удалена функция автоматического внесения добавленных ссылок в белый список для уведомлений и обратных ссылок, поскольку она допускала злоупотребление. (#13887)
  • Исправлено каноническое перенаправление постоянных ссылок с шаблоном %category% при наличии вложенных рубрик и разделения записи на страницы. (#13471)
  • Исправлено отображение неподходящих сообщений об ошибке в некоторых случаях при активации плагинов. (#15062)
  • Небольшие исправления XSS в request_filesystem_credentials() и при удалении плагинов. (r16367, r16373)
  • Уточнение лицензии в readme (r15534)
  • Multisite: Исправлена возможность delete_user (r15562)
  • Multisite: current_user_can_for_blog() вызывает map_meta_cap() даже для суперадминистраторов (#15122)
  • Multisite: Исправлены заголовки типа содержимого в ms-files.php при запросе файлов с параметрами в адресной строке (#14450)
  • Multisite: Исправлено использование константы SUBDOMAIN_INSTALL на обновлённых установках WordPress MU (#14536)
