On February 21, 2019, WordPress 5.1 “Betty”, named for the jazz musician Betty Carter, was released to the public. For more information on this release, read the WordPress Blog.

\n\n\n\n

For Version 5.1, the database version (db_version in wp_options) updated to 44719, and the Trac revision was 44764.

\n\n\n\n

A full list of tickets include in 5.1 can be found on Trac.

\n\n\n\n

Highlights # Highlights

\n\n\n\n
\n\n\n\n

Following the introduction of the new block editor in 5.0, 5.1 focuses on polish, in particular by improving performance of the editor. In addition, this release paves the way for a better, faster, and more secure WordPress with some essential tools for site administrators and developers.

The WordPress 5.1 release post,
\n\n\n\n

Site Health # Site Health

\n\n\n\n

With security and speed in mind, this release introduces WordPress’s first Site Health features. WordPress will start showing notices to administrators of sites that run long-outdated versions of PHP, which is the programming language that powers WordPress.

\n\n\n\n
\n\n\n\n

When installing new plugins, WordPress’s Site Health features whether a plugin requires a version of PHP incompatible with your site. If so, WordPress will prevent you from installing that plugin.

\n\n\n\n

Editor Performance # Editor Performance

\n\n\n\n

Introduced in WordPress 5.0, the new block editor continues to improve. Most significantly, WordPress 5.1 includes performance improvements within the editor. The editor should feel a little quicker to start, and typing should feel smoother.

\n\n\n\n
\n\n\n\n

Expect more performance improvements in the next couple of releases.
