FAQ Sobre Backups[edit]

Faça Backup do seu banco de dados regularmente e sempre antes de qualquer upgrade (atualização).


Quantas vezes se deve fazer backup? 

Isso depende de quantas vezes você edita seu blog, quantas vezes você quiser fazer isso, e como você se sentiria se o seu banco de dados fosse perdido além de todas as suas postagens. É sua decisão.

É possível fazer backup de outros dados? 

Sim. E você deve!

Quantos backups devo manter? 

A maioria das pessoas faz um backup e em seguida, apenas o substituí. Isso economiza espaço e não se tem com que se preocupar. Mas e se o arquivo de backup é corrompido ou perdido? E agora? A regra geral é manter pelo menos três backups e mantê-los em três lugares diferentes ou de outras formas, como disquetes, CDs, diferentes unidades de disco rígido, em um servidor remoto, sua conta de e-mail, etc

Meus arquivos de Backup são enormes! Posso fazer alguma coisa sobre isso? 

Possivelmente. Normalmente, anti-spam e plugins de estatísticas podem adicionar grandes quantidades de dados porque estão constantemente na recolha de informação, o banco pode inchar significativamente. Quanto ao backup do banco de dados, tais informações provavelmente não são importantes para manter. Não marque as tabelas de tais dados ou use o plugin para excluir estes dados armazenados.

Backups podem ser automatizados? 

Sim. Existem vários métodos de automatizar o processo de backup disponível, mas faça tambpem um backup manual de vez em quando para garantir que o processo está funcionando. Veja Recursos de Backup para mais informações.
