Habilite Cookies[edit]

Certifique-se que os cookies estão habilitados para o seu navegador.

  • Limpar os cookies do navegador.
  • Limpar o cache do navegador.

Desabilitar Plugins[edit]

Alguns WordPress Plugins podem interferir com o processo de login. Desative todos os Plugins, pode ser através do Painel de Administração ou de remova/renomeie sua pasta em /wp-content/plugins/.

Novo Arquivo de Login[edit]

Às vezes, o wp-login.php pode ter sido corrompido ou enviado incorretamente.

  • Delete wp-login.php e envie uma nova cópia de um pacote WordPress. Lembre-se, a substituição de arquivos via Clientes FTP pode levar à transferência incompleta.
  • Edite wp-login.php como segue:
  • Vá até a seção case retrievepassword, encontre o comentário:


// redefining user_login ensures we return the right case in the email

Substitua:

$user_login = $user_data["user_login"];

Com:

$user_login = $user_data->user_login;

Editando Tabela de Usuários[edit]

Este é um modo não-recomendando para iniciantes.

Accesse phpMyAdmin do seu servidor e edite o banco de dados com cautela:

  • Abra o banco de dados WordPress.
  • Clique na tabela de usuários no menu à esquerda (o padrão é wp_users , ou seuprefixo_users se você definiu um prefixo de tabela diferente.)
  • Clique em 'BROWSE'.
  • Clique no botão "EDIT'botão 'ao lado do usuário admin.
  • Delete o que estiver no campo de senha.
  • Utilizar o MD5 Hasher, digite a senha desejada e copie o hash gerado.
  • Cole o hash no campo de senha para o usuário admin .
  • Guardar a informação na base de dados.
  • Entrar no WordPress usando "admin" e a senha que você usou para criar o hash.
  • No caso de novas versões do WordPress, a senha é duplo hash, mas enquanto é md5 a senha, ele vai completar o hash (ele atualiza sua senha por você).

Problemas com senha[edit]

Para obter informações sobre problemas em iniciar sessão, devido a uma senha errada ou perdida, veja Redefinindo Senha.

Lembre-se, os campos Username e Password são sensíveis a letra maiúscula/minuscula.

URL Redirecionado[edit]

Em certos casos, o seu endereço WordPress URI pode ter sido redefinido.

  • Verifique o valor siteurl na tabela wp-options do banco de dados do WordPress.
  • Está definido como http:/?
  • Se sim, mudesite_url para o valor correto.
  • Abrawp-login.php em um edito de texto e comente/delete as seguinte linhas:

// If someone has moved WordPress let's try to detect it //if ( dirname('http://' . $_SERVER['HTTP_HOST'] . $_SERVER['PHP_SELF']) != get_settings('siteurl') ) 

// update_option('siteurl', dirname('http://' . $_SERVER['HTTP_HOST'] . $_SERVER['PHP_SELF']) );

HTTPS Seguro[edit]

Se você está tendo problema com HTTPS seguro, altere a seguinte linha em wp-includes/vars.php de:

define('COOKIEPATH', preg_replace('|http://[^/]+|i', , get_settings('home') . '/' ) );

Para:

define('COOKIEPATH', preg_replace('|https?://[^/]+|i', , get_settings('home') . '/' ) );

Certifique-se também de alterar o URL em Opções para https ://.
