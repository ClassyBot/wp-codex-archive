Depurando uma Rede WordPress[edit]

Se você chegou a esta página, é provável que você tenha recebido um erro em sua rede WordPress. Esta falha ocorre quando o WordPress não pode encontrar um ou mais das tabelas globais para a rede, no banco de dados (artigo em inglês).

Em alguns serviços compartilhados, o servidor desativa a funcionalidade de execução. É sempre melhor verificar com o seu serviço de hospedagem antes de criar a rede para garantir que seu serviço de hospedagem satisfaça os requisitos técnicos.

Se você acaba de instalar sua rede[edit]

Verifique seu arquivo wp-config.php:

  • informações corretas do banco de dados
  • constante SUBDOMAIN_INSTALL
  • constante MULTISITE
  • variável $base
  • prefixo da tabela
  • Você não deve ter nada depois de
/* That's all, stop editing! Happy blogging. */

/** Absolute path to the WordPress directory. */
if ( !defined('ABSPATH') )
	define('ABSPATH', dirname(__FILE__) . '/');

/** Sets up WordPress vars and included files. */
require_once(ABSPATH . 'wp-settings.php');
Mova qualquer código depois de 
require_once(ABSPATH . 'wp-settings.php');
acima da linha Isso é tudo! pare de Editar!.

Verificar o Banco de Dados[edit]

Assumindo que tudo está correto, verifique se o banco de dados e veja se todas as tabelas extras do banco de dados(artigo em inglês) foram criadas. As tabelas são:

  • wp_blogs
  • wp_blog_versions
  • wp_registration_log
  • wp_signups
  • wp_site
  • wp_sitemeta

Se a criação do novo site de repente parou de funcionar[edit]

Verifique seu banco de dados como descrito acima. Verifique se o lugar do servidor de banco de dados não mudou, e se for isso, atualize o seu arquivo wp-config.php.

Outros problemas menos conhecidos[edit]

Verifique se o usuário do banco de dados tem TODAS as permissões no banco de dados.

Além disso, em poucos upgrades de WordPressMU a 3,0 para cima, um pequeno número de usuários teve um problema com a criação de novos sites e receberam erros. Isto acabou por ser uma questão de agrupamento (collation) do banco de dados.

Verifique se as instruções. htaccess não está jogando erros nos logs do Apache.

Como este: 

Options FollowSymLinks or SymLinksIfOwnerMatch is off which implies that RewriteRule directive is forbidden:

Isso resultará em uma instalação de rede que parece falhar e pode mostrar erros como: 

One or more database tables are unavailable. The database may need to be repaired.
