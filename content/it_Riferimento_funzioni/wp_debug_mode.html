Descrizione[edit]

Imposta la gestione degli errori di PHP e gestisce la modalità di debug di WordPress.

Utilizza tre costanti: WP_DEBUG, WP_DEBUG_DISPLAY e WP_DEBUG_LOG. Tutte e tre possono essere definite in wp-config.php. Esempio: 

define( 'WP_DEBUG', true );

WP_DEBUG_DISPLAY e WP_DEBUG_LOG non svolgono alcuna funzione a meno che WP_DEBUG non sia true. WP_DEBUG ha come valore predefinito false.

Quando WP_DEBUG è true, tutte le notifiche PHP vengono riportate. Anche WordPress stamperà notifiche, inclusa quella riguardante l'eventuale uso di una funzione deprecata, un argomento deprecato o un file deprecato. Il codice deprecato potrebbe essere rimosso nelle versioni future.

Si raccomanda fortemente agli sviluppatori di plugin e temi di utilizzare WP_DEBUG nei loro ambienti di sviluppo.

Quando WP_DEBUG_DISPLAY è true, WordPress forza la stampa degli errori. WP_DEBUG_DISPLAY ha come valore predefinito true. Impostarlo a null impedisce a WordPress di cambiare l'impostazione di configurazione globale. Impostare WP_DEBUG_DISPLAY a false fa sì che gli errori vengano nascosti.

Quando WP_DEBUG_LOG è true, gli errori vengono salvati nel file wp-content/debug.log. WP_DEBUG_LOG ha come valore predefinito false.

Utilizzo[edit]

<?php wp_debug_mode(); ?>
