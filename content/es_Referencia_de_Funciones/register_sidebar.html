Descripción[edit]

Construye la definición para una sidebar y devuelve el ID. Debe ser llamada en la acción "widgets_init".

Uso[edit]

<?php register_sidebar( $args ); ?>

Uso por defecto[edit]

<?php $args = array(
	'name'          => __( 'Nombre de la sidebar', 'text_domain' ),
	'id'            => 'identificador-unico-sidebar',
	'description'   => 'Descripción de la sidebar',
        'class'         => '',
	'before_widget' => '<li id="%1$s" class="widget %2$s">',
	'after_widget'  => '</li>',
	'before_title'  => '<h2 class="widgettitle">',
	'after_title'   => '</h2>' ); ?>

Parámetros[edit]

args
(string/array) (opcional) Construye la sidebar con los valores de 'name' e 'id'.
Default: None
  • name - Nombre de la sidebar (por defecto es 'Sidebar' y el ID numérico).
  • id - Identificador de la sidebar - Debe estar en minúsculas, sin espacios (por defecto se asigna un ID numérico autoincremental). Si no estableces el valor del argumento id, obtendrás un mensaje E_USER_NOTICE en el modo debug a partir de la versión 4.2.
  • description - Descripción de qué hace la sidebar y dónde se muestra. Aparece en la pantalla de administración de widgets. (Desde 2.9) (Por defecto, vacío).
  • class - Clase CSS que se asignará a la sidebar en la página de administración Apariencia -> Widget. Esta clase sólo aparecerá en el código de la página de administración de widgets de WordPress. No se incluirá en el front-end de la web. Nota: El valor "sidebar" se añadirá antes del valor de la clase. Por ejemplo, una clase "tal" tendrá finalmente el nombre de clase de "sidebar-tal". (Por defecto, vacío).
  • before_widget - HTML a colocar antes de cada widget (por defecto: <li id="%1$s" class="widget %2$s">) Nota: usa sprintf para la sustitución de variables
  • after_widget - HTML a colocar después de cada widget (por defecto: </li>\n).
  • before_title - HTML a colocar antes de cada título (por defecto: <h2 class="widgettitle">).
  • after_title - HTML a colocar después de cada título (por defecto: </h2>\n).

El parámetro opcional args es un array asociativo que se pasará como primer argumento en cada llamada a una sidebar activa. (Si se pasa un string en lugar de un array, se pasará a través de parse_str() para generar un array asociativo.) El uso básico de estos argumentos es pasar etiquetas HTML específicas del tema para envolver el widget y su título.

Notas[edit]

  • Con WordPress 3.4.1 hay aún algunos IDs a evitar, que pueden consultarse aquí.
  • Llamar a register_sidebar() varias veces para registrar varias sidebars es preferible a usar register_sidebars() para crearlas de una vez, porque te permite asignar un nombre único a cada una (por ejemplo: “Sidebar derecha”, “Sidebar izquierda”). Aunque estos nombres sólo aparecen en la interfaz de administración, es una buena práctica dar un nombre específico a cada sidebar, de forma que el usuario tenga una idea de cómo y dónde se utilizará cada una.
