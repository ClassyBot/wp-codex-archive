Description[edit]

admin_page_access_denied is an action hook triggered when a user does not have permission to access a page in the control panel. This is most likely due to not having the required capability to access the page.

The action hook is fired before wp_die is executed, and is a result of the function user_can_access_admin_page returning false.

Parameters[edit]

There are no arguments passed to this hook.

Usage[edit]

<?php add_action( 'admin_page_access_denied', 'callback' ); ?>

Where "callback" is the name of a function, or method, to be executed.

Examples[edit]

<?php
add_action( 'admin_page_access_denied', 'my_plugin_log_permission_error');

function my_plugin_log_permission_error() {
    // your code here
}
?>
