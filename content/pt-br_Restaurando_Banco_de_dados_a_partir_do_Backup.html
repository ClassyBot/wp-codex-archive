Usando phpMyAdmin[edit]

phpMyAdmin é um programa usado para manipular bancos de dados remotamente através de uma interface web. Um bom pacote de hospedagem terá isto incluído. Para obter informações sobre como fazer backup do banco de dados do WordPress, consulte Backups do Banco de Dados.

As informações mostradas aqui foram testadas com phpMyAdmin 2.8.0.2 em Unix.

Processo de Restauração[edit]

Tela do phpMyAdmin, observe os dados listados nesta tela, eles podem ser úteis para preencher o arquivo wp-config.php
  1. Faça Login no phpMyAdmin.
  2. Se o banco de dados é novo, crie um novo.
  3. O novo banco de dados aparecerá listado no lado esquerdo. Clique no nome do banco de dados que criou
  4. Clique na aba Importar.
  5. Clique no botão Procurar que aparecerá. Encontre o arquivo em seu computador.
  6. Marque o itemSQL.
  7. Clique no botão Executar.

Agora, tome um café. Talvez leve algum tempo. Eventualmente, você verá uma tela de sucesso.

Se você receber uma mensagem de erro, sua melhor aposta é procurar ajuda no fóruns de suporte WordPress.

Usando Comandos Mysql[edit]

O processo de restauração consiste em extrair o arquivo de dados e importá-lo para o banco de dados Mysql.

1. Extraia o conteúdo do arquivo .zip: 

user@linux:~/files/blog> bzip2 -d blog.bak.sql.bz2

Nota: Se o arquivo de backup for do tipo .tar.gz chamado blog.bak.sql.tar.gz então use o comando tar -zxvf blog.bak.sql.tar.gz em vez do descrito acima.

2. Injete o backup para o banco de dados: 

user@linux:~/files/blog> mysql -h mysqlhostserver -u mysqlusername
 -p databasename < blog.bak.sql

Informe a senha do MySQL:
user@linux~/files/blog:>
