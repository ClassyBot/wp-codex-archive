Beschreibung[edit]

Diese Funktion lädt die übersetzten Zeichenketten des Plugins.

Wenn kein Pfad definiert wird, dann wird die Wurzel des Plugin-Verzeichnisses als Pfad angenommen. Die Bezeichung der .mo-Datei sollte durch die Textdomäne des Plugins, gefolgt von einem Bindestrich und dann schließlich durch den Gebietsschema definiert werden.

Zur Veranschaulichung: Das Gebietsschema für Deutsch ist 'de_DE' und das Gebietsschema für Dänisch ist 'da_DK'. Würde man nun ein Plugin mit der Textdomäne "my-plugin" ins deutsche übersetzen wollen, dann müssen die .mo und .po Datein wie folgt benannt werden: "my-plugin-de_DE.mo" und "my-plugin-de_DE.po". Die Funktion sollte bereits bei der init aktion aufgerufen werden.

Wenn man load_plugin_textdomain mehrmals für die dieselbe Textdomäne aufruft, dann werden die Übersetzungen zusammengeführt. Wenn beide Mengen dieselbe zu übersetzende Zeichenkette beinhalten, dann wird die Übersetzung aus der Menge die zuerst aufgerufen wurde übernommen.

Verwendung[edit]

<?php load_plugin_textdomain( $domain, $abs_rel_path__DEPRECATED, $plugin_rel_path ) ?>

Parameter[edit]

$domain
(string) (notwendig) Eindeutiger Bezeichner für das Abrufen übersetzter Zeichenketten.
Default: Keins
$abs_rel_path (Veraltet)
(string) (optional) Relativer Pfad zu ABSPATH eines Verzeichnises, wo die .mo-Datei sich befindet. Veraltet, aber noch funktionsfähig bis Version 2.7
Default: false
$plugin_rel_path
(string) (optional) Relativer Pfad zu WP_PLUGIN_DIR. Dies ist der bevorzugte Parameter und sollte anstelle von $abs_rel_path verwendet werden
Default: false

Rückgabewerte[edit]

boolean 
Liefert true zurück wenn entweder der Filter override_load_textdomain true zurückgibt oder die Sprachdatei erfolgreich geladen wurde. Es liefert false zurück wenn die Sprachdatei nicht geladen werden konnte (sie ist nicht lesbar oder der mo-Datei-Reader kann sie nicht verstehen)..

Beispiele[edit]

Dieses Beispiel geht davon aus, dass die Datei mit den übersetzten Zeichenketten sich in den "languages"-Verzeichnis befindet. Falls dies nicht der Fall ist, dann soll man den Relativen Pfad entsprechend anpassen. 

/**
 * Plugin Textdomäne laden.
 *
 * @seit 1.0.0
 */
function myplugin_load_textdomain() {
  load_plugin_textdomain( 'my-plugin', false, basename( dirname( __FILE__ ) ) . '/languages' ); 
}

add_action( 'init', 'myplugin_load_textdomain' );

Anmerkungen[edit]

  • l10n ist eine Abkürzung für Lokalisierung.
  • l10n ergibt sich aus den 10 Buchstaben zwischen dem "l" und dem "n" von localization.

Änderungsprotokoll[edit]

  • Seit: 1.5.0
  • 2.7.0: '$abs_rel_path' parameter wurde als veraltet markiert.

Quelldatei[edit]

load_plugin_textdomain() befindet sich in wp-includes/l10n.php.
