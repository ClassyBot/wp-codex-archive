FAQ sur comment combattre les spams de commentaires[edit]

Pourquoi m’envoie-t-on des spams[edit]

Si ce n'est parce qu'ils le peuvent, le besoin de gagner des places dans le classement des pages des moteurs de recherche conduit au besoin de créer de nombreux liens vers une page de sorte que le moteur de recherche pense que cette page est importante.

Où puis-je trouver des plugins WordPress anti-spam[edit]

Les liste des plugins et outils anti-spam de Official WordPress Plugin Directory (anglais).

Quel est le meilleur plugin ?[edit]

Il n’y a pas de réponse facile. Les spammeurs utilisent différentes tactiques et chaque plugin va utiliser des tactiques différentes pour contrer les spams. Il n’existe pas de plugin qui vous protégera pour toujours de tous les spams. Les spammeurs de commentaires peuvent télécharger les plugins, analyser leur fonctionnement et trouver des façons de les contourner. C’est pourquoi, vous devriez surveiller les mises à jour sur le site de l’auteur du plugin. WordPress offre le moyen d’accéder facilement aux mises à jours via le Panneau de Plugins et le lien vers le site internet de l’auteur du plugin.

J’ai un problème avec un plugin[edit]

Visitez le site de l’auteur du plugin pour voir si une solution à votre problème n’y est pas déjà offerte. Si non, faites d’abord une recherche dans les forums avant de poser votre question car la réponse pourrait déjà être là.

Qu’arrive-t-il aux commentaires qui sont marqués "Spam"?[edit]

Les commentaires marqués [spam] seront conservés dans votre base de données pour éduquer les plugins anti-spams intelligents tels que Akismet (anglais). Les plugins comme Spam Nuker (anglais) peuvent restaurer de façon sûre ou effacer de manière permanente tout commentaire marqué [spam].

Un spam apparaît dès que je poste![edit]

Si vous n’avez pas de plugins pour gérer les spams de commentaires ou la dernière version de WordPress installée Dernière Version de WordPress (anglais), cela peut arriver. Mettez à Jour WordPress ou installez un des plugins anti-spam de commentaires et effacer manuellement les spams de commentaires dans votre panneau d’administration Gérer > Commentaires.

Pourquoi chaque commentaire va dans la file d’attente de modération ?[edit]

Allez au panneau Options > Discussion et assurez vous que l’option Un administrateur doit approuver le commentaire (qu’importe les choix ci- dessous) n’est pas cochée. Lorsque cette option est sélectionnée, tous les commentaires sont envoyés dans la file d’attente de modération en attente d’une approbation. Assurez vous que l'option Garder un commentaire dans la file s'il contient plus de x liens contient bien une valeur supérieure à zéro et n’est pas vide. Si le champ est vide ou la valeur égale à zéro, tous les commentaires contenant des liens seront modérés. Si l’option mentionnée précédemment n’est pas cochée, la valeur pour les liens plus grande que zéro, et si vous avez encore ce problème, votre Liste Noire de Mots de Spam Courants contient probablement des lignes vides, des signes de ponctuation, ou des lettres isolées entre les éléments de la liste. Il devrait y avoir des espaces entre les items de la liste, ou chaque item doit être sur une seule ligne. Si vous avez fait tout cela, alors faites une mise à jour des plugins anti-spam que vous avez installés. Si le problème persiste, désactivez un à un les plugins anti-spam de commentaires pour trouver le coupable et contactez l’auteur du plugin en question pour avoir son aide.

J’ai désactivé les commentaires mais j’en reçois encore[edit]

Si vous n’avez pas coché l’option Autoriser les commentaires sur cet article dans le panneau d’administration Options > Discussion, alors vous avez seulement désactivé les commentaires pour les futurs articles. Pour désactiver totalement les commentaires, vous allez devoir éditer chaque article passé et décocher Autoriser les commentaires dans le sous-panneau Écrire un Article (anglais). Ou bien, vous pouvez effacer le fichier wp-comments-post.php ou exécuter la requête MySQL qui suit, à partir de la ligne de commande sur votre compte shell, ou utiliser phpMyAdmin (anglais): UPDATE wp_posts SET comment_status="closed";

J’ai désactivé les trackbacks, mais des trackbacks continuent d’être envoyés[edit]

Si vous avez décoché l’option Autoriser les notifications depuis les autres weblogs (pingbacks et trackbacks) dans le panneau d’administration Options > Discussion, alors vous avez uniquement désactivé les trackbacks pour les futurs articles. Pour complètement désactiver les trackbacks, vous allez devoir éditer chaque article passé et décocher l’option Autoriser les Pings dans le sous-panneau Écrire un Article. Ou bien, Vous pouvez simplement effacer le fichier wp-trackback.php, ou exécuter la requête MySQL qui suit, à partir de la ligne de commande sur votre compte shell, ou utiliser phpMyAdmin (anglais): UPDATE wp_posts SET ping_status="closed";

Très utiles ces outils anti-spam?[edit]

Oui, ils sont très utiles. Alors, n'hésitez pas à laisser un petit mot de remerciement sur la page de l'auteur du plugin. Ces auteurs méritent des encouragements et aiment avoir des suggestions et des retours d'expérience sur leur travail. Cela les encouragera à continuer le développement de leur(s) plugin(s).
