Scopo della classe WP_List_Table[edit]

Questa classe viene usata per generare le List Tables che si trovano in tante pagine del pannello di amministrazione di WordPress. Il vantaggio rispetto alle precedenti implementazioni è che può essere modificata dinamicamente con AJAX e potrebbe essere agganciata con hooks nelle future versioni di WordPress.

Uso da parte degli sviluppatori e Classificazione privata[edit]

Il 27 Marzo 2012, Andrew Nacin ha avvertito gli sviluppatori che questa classe è stata creata solo per uso privato da parte del core e che potrebbe essere sottoposta a modifiche nelle versioni future di WordPress.

Nonostante ciò, la classe WP_List_Table è stata largamente usata dagli sviluppatori di WordPress e di Plugin poiché fornisce un modo affidabile e semanticamente consistente per creare liste tabellari personalizzate in WordPress. Nel corso del tempo non sono stati fatte modifiche importanti a questa classe e nemmeno si prevede di farne, quindi testare i tuoi Plugin con la versione beta/RC di WordPress, dovrebbe essere più che sufficiente per evitare problemi.

Se non desideri assumerti questo rischio, un sistema comune (e sicuro ) per aggirare il problema è quello di fare una copia della classe WP_List_Table ( /wp-admin/includes/class-wp-list-table.php ) da usare e distribuire con il tuo Plugin invece di usare la classe originale contenuta nel core. In questo modo anche se la classe contenuta nel core cambia, il tuo plugin continua a funzionare.

In ogni caso gli sviluppatori che desiderano usare questa classe lo fanno a proprio rischio.

Utilizzo[edit]

Questa classe è stata pensata per essere usata come una specie di framework, essendo che i dati devono essere caricati, ordinati e filtrati manualmente. Nonostante ciò è potenzialmente un tool molto potente per gli sviluppatori dato che crea liste tabellari standard di WordPress che rendono facile implementare funzioni avanzate come paginazioni, azioni, azioni cumulative e filtri.

Per usare WP_List_Table, prima di tutto devi creare una classe derivata da quella originale. La nuova classe deve essere istanziata ed i metodi prepare_items() e display() dovranno essere chiamati esplicitamente. Per i dettagli leggi la descrizione dei metodi più avanti.

Nel Core di WordPress[edit]

Il core di WordPress carica e restituisce le proprie classi dinamicamente usando la funzione _get_list_table() che automaticamente carica la giusta classe derivata e la istanzia. Questa comunque è una funzione privata e non dovrebbe essere usata dagli sviluppatori.

Uso da parte degli sviluppatori[edit]

Siccome questa classe è marcata come privata, gli sviluppatori la usano a proprio rischio perché potrebbe essere sottoposta a cambiamenti nelle prossime versioni di WordPress. Ogni sviluppatore che la usa viene fortemente incoraggiato a testare i propri plugin con la versione beta/RC di WordPress per mantenere la compatibilità.

Poiché i sviluppatori non possono usare la funzione _get_list_table() direttamente, la classe deve essere estesa ed istanziata manualmente ad esempio in questo modo... 

class Example_List_Table extends WP_List_Table {}
$example_lt = new Example_List_Table();

Questo esempio non visualizza niente, si DEVONO aggiungere alcuni metodi ed i dati per rendere la WP_List_Table utilizzabile.

Metodi e Proprietà[edit]

Proprietà[edit]

Le seguenti proprietà sono integrate nella classe base WP_List_Table. Nota che i metodi magici __get e __set di WP_List_Table impediscono di impostare variabili istanza diverse nelle sottoclassi di WP_List_Table, per favore usate le variabili raccomandate:

$items
Questa viene usata per memorizzare i dati grezzi che vuoi visualizzare. Di solito i dati vengono caricati direttamente con il metodo prepare_items().
$_args
Contiene varie informazioni sulla tabella corrente (in formato array). di solito non viene manipolata direttamente.
$_pagination_args
Contiene informazioni necessarie per gestire la paginazione (in formato array). Di solito non viene manipolata direttamente, ma utilizzando i comandi get_pagination_arg( string ) o set_pagination_args(array).
$screen
Questo viene usato per memorizzare la videata corrente, quando è necessario per renderla consistente con la intera istanza.
$_actions
Contiene la cache delle azioni cumulative. Solitamente non viene manipolata direttamente.
$_pagination
Contiene la cache della paginazione. Solitamente non viene manipolata direttamente.

Proprietà estese[edit]

Queste proprietà non solo integrate nella classe principale, ma sono attese da alcuni metodi. Devono essere definite manualemente nelle classi derivate.

$_column_headers
Nel core questa proprietà viene assegnata automaticamente. Gli sviluppatori devono definirla manualmente nei propri metodi prepare_items() o __construct().
Questa proprietà richiede un array con 4 valori :
  • Il primo valore contiene gli slug ed i titoli delle colonne (vedi il metodo get_columns() ).
  • Il secondo valore è un array che contiene i valori del campi da nascondere.
  • Il terzo valore è un array di colonne che abilità l'ordinamento (vedi il metodo get_sortable_columns()).
  • Il quarto valore è una stringa che indica quale colonna utilizzare come primaria per visualizzare le azioni di riga (edit, visualizza, ecc.). Il valore deve corrispondere con uno degli slug di colonna del primo valore.

Metodi[edit]

__construct( $args=array() )
Il Costruttore. Questo imposta gli argomenti ed i filtri di default. Gli sviluppatori dovrebbero ridefinirlo, chiamado il costruttore parent per fornire valori per singolari e plurali e per consentire alla classe di supportare AJAX.

-; ajax_user_can()

Può essere sovrascritta per attivare alcune funzionalità di permessi alla tua tabella.
prepare_items()
Gli sviluppatori devono usare questo metodo per filtrare i dati, gestire ordinamenti, paginazioni e ogni altro tipo di manipolazione dei dati da fare prima del rendering. Questo comando deve essere chiamato esplicitamente dopo l'istanziazione della classe e prima del rendering.
set_pagination_args( $args )
Questo metodo deve essere chiamato internamente (di solito da prepare_items()) per impostare gli argomenti di base della paginazione che sono:
  • total_items - il numero totale di elementi da visualizzare. Di solito semplicemente count($data)
  • per_page - il numero di elementi per pagina
  • total_pages - il nunero di pagine. Può essere lasciato in bianco o calcolato manualmente ad esempio: ceil($total_items/$per_page)
get_pagination_arg( $key )
Restituisce il singolo argomento di paginazione identificato dalla chiave.
has_items()
Indica se la proprietà items è vuota.
no_items()
Restituisce il messaggio da visualizzare quando items è vuoto.
search_box( $text, $input_id )
Questo prepara una casella di ricerca, però è necessario inserire manualmente la tabella (compresa la casella di ricerca) in una form.
get_views()
Restituisce un array associativo che elenca tutte le viste che possono essere usate con la tabella.
views()
Prepara l'elemento <ul> che contiene i nomi delle viste.
get_bulk_actions()
Sovrascrivi questo metodo per restituire un array associativo nella forma ( action_slug => action_title ) contenente tutte le azioni cumulative disponibili per la tabella.
bulk_actions()
Quando viene chiamato prepara un drop-down con le azioni cumulative, però è necessario inserire manualmente la tabella (compresa la casella di ricerca) in una form.

will still need to manually wrap your list table (including search box) in a form.

current_action()
Restituisce l'azione corrente selezionata con il drop-down delle azioni cumulative.
row_actions( $actions, $always_visible = false )
Chiama questo metodo (di solito da uno dei metodi di colonna) per inserire la div con le azioni di riga. il parametro $actions deve essere un array associativo in cui la chiave è il nome della azione ed il valore è un link.
months_dropdown( $post_type )
Chiama questo metodo per preaparare un drop-down per le date, per filtrare gli items per mese.
view_switcher( $current_mode )
Chama questo metodo per preparare un pulsante per passare alla view post-type oppure (List View e Excerpt View)
comments_bubble( $post_id, $pending_comments )
Chiama questo per preparare un pop-up di tipo commento.
get_pagenum()
Restituisce il numero di pagina corrente.
get_items_per_page( $option, $default = 20 )
Restituisce il numero di items da visualizzare nella pagina, come scelto dagli utenti.
pagination( $which )
Crea l'HTML di paginazione e lo assega alla proprietà _pagination. Di solito non è necessario chiamare questo metodo perché lo fa per noi il metodo display().
get_columns()
Questo metodo deve essere sovrascritto per restuire un array associativo di colonne. L'array associativo deve essere nella forma 'slug' => array( 'Title', true ). Tipicamente userai questo nel metodo prepare_items() per costurire parte della proprietà _column_headers.
get_sortable_columns()
Questo metodo deve essere sovrascritto per restituire un array associativo delle colonne che vuoi rendere ordinabili. L'array associativo deve essere nella forma 'column_slug'=>array('sortby',true), in cui il secondo parametro indica se il campo è preordinato. Questo si usa spesso per fornire una parte della proprità _column_headers.
get_column_info()
Viene usato da WordPRess per costruire la proprietà _column_headers. Di solito non dovrebbe essere usato dagli sviluppatori. Invece si deve assegnare la proprietà _column_headers direttamente dal metodo prepare_items().
get_column_count()
Restituisce semplicemente il numero delle colonne visibili.
print_column_headers( $with_id = true )
Questo metodo prepara le intestazioni delle colonne. Di solito non è necessario chiamare questo metodo perché lo fa per noi il metodo display().
display()
Questo comando si usa per inviare tutta la tabella alla pagina.
get_table_classes()
Restituisce la lista delle classi css da aggiungere a tutti gli elementi della tabella. Sovrascrivi per personalizzare le classi della tabella.
display_tablenav( $which )
Questo genera i comandi di navigazione sopra o sotto la tabella. Di solito non è necessario chiamare questo metodo perché lo fa per noi il metodo display().
extra_tablenav( $which )
Questo metodo può essere sovrascritto per visualizzare controlli addizionali tra le azioni cumulative ed i controlli di navigazione.
display_rows_or_placeholder()
Questo genera la parte tbody della tabella. Di solito non è necessario chiamare questo metodo perché lo fa per noi il metodo display().
display_rows()
Questo itera gli elementi dalla proprietà items e li invia alla pagina in forma di righe della tabella. Di solito non è necessario chiamare questo metodo perché lo fa per noi il metodo display().
single_row( $item )
Questo invia alla pagina un singolo elemento (dalla proprietà items). Di solito non è necessario chiamare questo metodo perché lo fa per noi il metodo display().
single_row_columns( $item )
Questo prepara tutte le colonne per una singola riga. È importante capire che questo metodo si aspetta l'esistenza i alcuni metodi di colonna(es column_mycolumn()) e/o un metodo column_default(). Nessuno di questi viene fornito dalla classe base e deve essere scritto nella classe derivata. Di solito non è necessario chiamare questo metodo perché lo fa per noi il metodo display().
ajax_response()
Elabora le richieste AJAX in arrivo.
_js_vars()
Invia le variabili Javascript che vengono generate dinamicamente dalla classe.

Metodi estesi[edit]

Questi metodi non sono inclusi nella classe base ma possono, e dovrebbero, essere definiti nelle tue classi derivate!

column_default( $item, $column_name )
Questo è il metodo che viene usato per costruire una colonna quando non esiste un altro metodo specifico per quella colonna. Quando WP_List_Tables prova a costruire la colonna (in single_row_columns()), prima cerca un metodo specifico per la colonna. Se non esiste usa questo

metodo. Questo metodo accetta due argomento, un singolo array $item e il $column_name (il suo slug).

ATTENZIONE: Dalla versione WordPress 3.5.1, nel core il $item viene passato come Object, e non come array.
column_$custom( $item )
Le colonne personalizzate devono essere fornite dallo sviluppatore e possono essere usate per gestire ogni tipo di colonna individualmente. Per esempio: se viene fornito un metodo di none column_movie_title(), verrà usato per preparare qualsiasi colonna che ha lo slug "movie_title". Questa funzione accetta come argomento un singolo array $item.
ATTENZIONE: Dalla versione WordPress 3.5.1, nel core il $item viene passato come Object, e non come array.

Usare nei Meta Boxes[edit]

Attenzione Quando si usa la classe WP_List_Table dentro ad un Meta Box:

Se non vieme impostato $this->_column_headers nella propria classe con il metodo prepare_items() (come ad esempio nel blocco di codice che segue), allora tutto il <thead> e <tfoot> scompare! 

$this->_column_headers = array( 
	 $this->get_columns(),		// columns
	 array(),			// hidden
	 $this->get_sortable_columns(),	// sortable
);

Come mai succede?[edit]

Se non imposti $this->_column_headers; manualmente nella classe estesa, le colonne ordinabili vengono impostate dalla classe parent e dalla funzione $this->get_columns();.

Sfortunatamente, quando succede l'intestazione della tua colonna è vuota anche se le hai definite con il metodo get_columns(); della tua classe derivata.

La soluzione[edit]

Usa il filtro $GLOBALS['wp_filter']["manage_{$GLOBALS['screen']->id}_screen_columns"]; per recuperarli nuovamente!

Nota collaterale: La funzione get_column_headers() che applica questo filtro è definita in /core_root/wp-admin/includes/template.php

'C'è un problema importante:' Facendo questo, nessun altro meta box può contenere una estensione per la classe WP_List_Table, perché questo romperebbe le colonne dei meta box addizionali perché si può usare solo un filtro per le colonne per una videata.

Versione[edit]

WP_List_Table è stata implementata nella versione WordPress 3.1.
