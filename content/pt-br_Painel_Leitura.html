Configurações→ Leitura[edit]

As opções no SubPainel Leitura são em número reduzido, mas ainda assim importantes. Você pode decidir se quer uma página estática como página inicial. Você também pode ajustar a quantidade de posts que são exibidas na página principal. Além disso, você pode ajustar recursos do Feed para determinar como as informações do seu site são enviadas para um leitor web ou outras aplicações.

Configurações de Leituras[edit]

A página inicial mostra
Use essa configuração para determinar se sua página inicial mostra seus post recentes ou uma página "estática". Esta configuração exibe somente se você tiver uma ou mais páginas definidas. Saiba que alguns plugins podem afetar o funcionamento dessa opção.
  • Seus posts mais recentes - Verifique o botão de rádio para que seus mais recentes posts sejam exibidos na página inicial do blog. Lembre-se, o número de posts que vai mostrar, é controlado pelo valor em As páginas do blog mostram no máximo:.
  • Uma página estática (selecionar abaixo) - Marque este botão de rádio para fazer com que uma página "estática" seja mostrada como página inicial do seu blog. Ao mesmo tempo, escolha uma página para mostrar seus posts. A primeira página e a página de posts não podem ser as mesmas.
  • Página inicial - Selecione no menu suspenso, a página que você quer usar para ser sua página inicial. Se você não selecionar uma opção aqui, então efetivamente o seu blog vai mostrar seus posts em ambas as páginas do blog nas páginas de posts que especificar. Se você gostaria de criar um arquivo de modelo estático não use home.php como nome de arquivo, senão você vai encontrar problemas ao tentar visualizar o blog/posts do seu site. Para contornar esta situação, use qualquer outro nome de arquivo, por exemplo, paginainicial.php
  • Página de posts - Selecione no menu suspenso o nome da Página, que agora irá conter seus posts. Se você não selecionar uma página aqui, seus posts só serão acessíveis através de recursos de navegação, tais como calendário, categoria, ou links para arquivamento. Mesmo se a página selecionada for protegida por senha, aos visitantes não será solicitada uma senha ao exibir a página. Além disso, qualquer modelo atribuído a página será ignorado e o arquivo do tema index.php (ou home.php se existir) irá controlar a exibição dos posts.
Veja Creating a Static Front Page para mais detalhes.
As páginas do blog mostram no máximo
  • [X] posts - Digite o número de posts a serem exibidos, por página, em seu site.
Os feeds RSS mostram os últimos 
  • [X] posts - Digite o número de posts que as pessoas vão ver quando obter qualquer feed do seu site.
Para cada post em um feed, mostrar 
Determina se o feed mostra o post completo ou apenas um resumo.
  • Texto completo - Marque este botão de rádio para incluir o conteúdo integral de cada post
  • Resumo - Marque este botão de rádio para incluir o resumo de cada post. Isso economiza tráfego do seu servidor.
Visibilidade nos mecanismos de busca
Marque a opção Evitar que mecanismos de busca indexem este site para impedir a indexação do site e rastreio por mecanismos de busca mas permitir visitantes normalmente. Observe que cabe aos mecanismos de busca atender esta solicitação.

Salvar Alterações[edit]

Clique no botão Salvar Alterações para fixar qualquer alteração que fizer. Após clicar no botão, uma tela aparecerá confirmando que a modificação foi salva.
