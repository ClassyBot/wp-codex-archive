文档[edit]

Codex 上的所有文档正是由像你我这样的志愿者撰写的！若希望参与编辑 Codex 文档，请浏览这份列表。

WP Docs 邮件列表是用来讨论开发 WordPress 文档事项的区域。文档项目的目标可在关于 Codex 上获知。

此外，您也可以帮助翻译 Codex（参见下面章节）。

翻译[edit]

翻译 WordPress 介绍了翻译 WordPress 程序的流程，这样用户就可使用非英语语言来运行 WordPress 了。它也介绍了如何找寻并加入翻译 WordPress 和 Codex 文档。

wp-polyglots 邮件列表是讨论 WordPress 翻译相关问题和建议的区域，欢迎大家随时加入！

WordPress 支持论坛[edit]

此外，帮助其它用户使用 WordPress 也是协助开发的一种方式。WordPress 相关支持都可以在官方中文支持论坛上免费获得，同时，您的参与对支持论坛的维持也至关重要。

如有兴趣，请首先阅读支持论坛志愿者以了解如何协助支持论坛运行，并加入WP 论坛邮件列表。

WordPress IRC 在线聊天[edit]

WordPress 用户还可通过 IRC 寻求帮助和支持。在 IRC FreeNode 服务器下的 #wordpress 部分有 IRC 可讨论 WordPress 话题。您也可以在上面回答问题。更多信息，请查看 IRC 和 WordPress IRC 在线帮助。

WordPress 开发[edit]

无论您是否是软件开发人员，都可协助开发 WordPress 软件。

开发计划[edit]

参与 WordPress 开发的一个途径就是建议可开发的功能，参与开发计划过程。即使你不是开发人员，你也可以帮助总结民意，使提议更加正式，精简较长邮件和 IRC 聊天记录主题以使开发人员能很快确定 WordPress 用户希望拥有的新功能。

测试 WordPress[edit]

测试 WordPress 也是 WordPress 用户可以协助的事项之一。每个版本在正式发布之前，都会先测试一番。你可下载预发布版本并测试，这样 WordPress 开发人员在正式发布之前就可以修复 bug。如有兴趣，可加入 WP 测试者邮件列表 —— 公布新版本发布并进行讨论的区域。

如果您想紧跟开发信息，甚至在预发布前就了解信息，可在使用 Subversion 中的 WordPress Subversion（SVN）仓库查找最新软件。或者您也可以通过 http://wordpress.org/nightly-builds/wordpress-latest.zip 获取“每夜构建”（SVN 库创建的压缩文件）——几乎和 Subversion 仓库同时更新。

测试预发布版本或已正式发布版本时，如发现 bug，可进行以下操作：

报告 Bug[edit]

即通过报告 WordPress 软件中的 bug 来协助开发 WordPress。首先，请阅读 bug 报告，了解如何报告 bug 后，就可搜索WordPress Trac 数据库来查看是否已有人上报这个 bug，如果没有，就可以提交新报告了。我们非常感谢您提交的 bug 报告！

修复 Bug 与开发其他程序[edit]

如果您是软件开发人员，您也可以通过修复由您或他人提交到 WordPress Trac 数据库的问题来协助开发 WordPress，这就是通常所说的“补丁”，更多介绍，请查看 Bug 报告。

你也可以订阅wp-hackers 邮件列表以参与有关 WordPress 开发的事项。邮件列表欢迎大家的参与，意见和建议。有时，也有成员在上面寻求志愿者帮助以改进或开发 WordPress 的某个功能。

#wordpress-bugs IRC 频道有时也会举办 bug 日，在 WordPress Bug Hunts 可了解 bug 日将进行的活动。此外，订阅 wp-hackers 或 wp-testers 邮件列表都可获知 bug 日举办日期。

如有兴趣，可继续阅读代码提交 —— 详述了 WordPress 新版本如何修复 bug，创建代码及开发新特征。

添加链接[edit]

如果您喜欢 WordPress 或认为它真的实用，请您把 WordPress 链接放到您的网站上。您可以链接文本，也可使用 WordPress.ORG 上的任何一个WordPress 标志。记得要把您对WordPress 的喜爱付诸于实践哦！

捐助[edit]

WordPress 社区的存在正是由于大家都以一定的形式参与了进来，并奉献了各自的时间，精力，有时甚至是金钱，因为他们相信WordPress 提供的珍贵服务。我们也邀请您参与您认为合适的社区建设，并从为 WordPress 主题和插件作者募捐开始做起，因为正是这些作者为 WordPress 的广大用户免费贡献了自己的创意和专业技术。

如果你在使用 WordPress 主题或插件，或您的 WordPress 博客建立在其上，现在就可联系作者并询问你如何能给予回报以支持他们的工作。由于主题和插件必须随着 WordPress 的更新和 bug 的发现而更新或作修改，因此，创建并支持主题和插件会花费很多时间和精力。许多作者若发现你在使用他们的插件或主题都会非常高兴，他们一般也都会接受捐款。有些作者把他们的主题和插件视为实践组合（experiential portfolios）——用户可以试用它们，若是喜欢就可直接使用。许多作者也都明确表示如果用户能对他们的辛苦劳作给予回报，他们将不胜感激——回报他们就是回报 WordPress！

WordPress 社区如能给予程序员，开发人员，测试员与挑战者更多的支持，WordPress 就会变得更强更完善！募捐是支持的一种表达方式，此外，表达谢意也是一种支持方式！

请记住，大家的每份贡献都是不可或缺的。WordPress 的美好明天需要我们每个人的协助！
