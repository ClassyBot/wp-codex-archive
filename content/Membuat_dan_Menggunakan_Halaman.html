In WordPress, Anda dapat menulis baik posting atau Pages. Ketika Anda sedang menulis entri blog biasa, Anda menulis posting. Posting, dalam setup default, muncul dalam urutan kronologis terbalik di halaman rumah blog Anda. Halaman untuk konten seperti "Tentang," "Hubungi," dll halaman tinggal di luar kronologi blog normal, dan sering digunakan untuk menyajikan informasi yang abadi tentang diri Anda atau situs Anda - informasi yang selalu berlaku. Anda dapat menggunakan Pages untuk mengatur dan mengelola konten apapun.

Selain umumnya diperlukan "Tentang" dan "Kontak" Pages, contoh-contoh lain dari halaman umum termasuk Hak Cipta, Keterbukaan, Informasi Hukum, Reprint Izin, Informasi Perusahaan, dan Pernyataan Aksesibilitas.

Secara umum, Pages sangat mirip Entri dalam bahwa mereka berdua memiliki Judul dan Konten dan dapat menggunakan file WordPress Tema template Anda untuk mempertahankan tampilan yang konsisten di seluruh situs Anda. Halaman, meskipun, memiliki beberapa perbedaan kunci yang membuat mereka sangat berbeda dari Posting.

'Apa Pages Apakah:'

  • Pages adalah untuk konten yang kurang waktu tergantung dari Tulisan.
  • Artikel dapat diatur ke dalam halaman dan subhalaman.
  • Artikel dapat menggunakan berbagai Page Template yang dapat mencakup Template Files, Template Tags dan kode PHP lainnya.
  • Pages mungkin memiliki array yang lebih kompleks dari penyesuaian tampilan tersedia saat menggunakan canggih Tema dengan kustomisasi yang luas.
  • Pada dasarnya, Pages adalah untuk konten non-blog. Hal ini dimungkinkan untuk menghapus semua atau sebagian Posting dari instalasi WordPress, dan dengan demikian membuat website non-blog standar.

'Apa Halaman yang Tidak:'

  • Artikel tidak Posts, mereka juga tidak dikutip dari karya yang lebih besar dari fiksi. Mereka tidak melakukan siklus melalui halaman utama blog Anda. Plugin WordPress yang tersedia untuk mengubah default jika diperlukan.
  • Artikel tidak dapat dikaitkan dengan Kategori dan tidak dapat ditugaskan Tags. Struktur organisasi untuk Halaman datang hanya dari hubungan timbal balik hirarkis mereka, dan bukan dari Tag atau Kategori.
  • Artikel tidak file. Mereka disimpan dalam database Anda seperti Posts berada.
  • Meskipun Anda dapat menempatkan Template Tag dan PHP kode ke file Template Page, Anda tidak dapat menempatkan ini ke dalam Page atau konten Pasang tanpa Plugin WordPress seperti PHP yang menimpa proses penyaringan kode.
  • Halaman tidak termasuk dalam feed situs Anda.
  • Pages dan Tulisan dapat menarik perhatian dengan cara yang berbeda dari manusia atau mesin pencari.
  • Pages (atau pos tertentu) dapat diatur sebagai halaman depan statis ". Blog" jika diinginkan dengan Halaman terpisah ditetapkan untuk posting blog terbaru, biasanya bernama

Membuat Halaman[edit]

kanan | thumbnail | Pages> Add New Untuk membuat Halaman baru, login ke instalasi WordPress Anda dengan hak admin cukup untuk membuat artikel baru. Pilih Pages> Add New pilihan untuk memulai menulis Halaman baru.


Pengorganisasian Halaman Anda[edit]

Sama seperti Anda dapat memiliki subkategori dalam Kategori Anda, Anda juga dapat memiliki subpages dalam Pages, menciptakan hirarki halaman.

Sebagai contoh, sebuah situs Wordpress untuk agen perjalanan dapat menampilkan sebuah halaman individu untuk setiap benua dan negara yang badan dapat membuat pengaturan perjalanan. Di bawah Halaman berjudul "Afrika" akan Subpages untuk Lesotho, Kamerun, Togo, dan Swaziland. Orang tua lain Halaman "Amerika Selatan" akan menampilkan Subpages dari Brasil, Argentina, dan Chile. Situs Anda akan daftar:

  • Afrika
    • Kamerun
    • Lesotho
    • Swaziland
    • Untuk pergi
  • Amerika Selatan
    • Argentina
    • Brazil
    • Chili

Untuk membuat subhalaman[edit]

  1. Pergi ke Administrasi> Pages> Add New panel.
  2. Pada menu kanan, klik "Halaman Parent" menu drop-down. Menu drop-down berisi daftar semua Pages sudah dibuat untuk situs Anda.
  3. Pilih yang tepat orangtua Halaman dari menu drop-down untuk membuat Halaman arus Halaman anak.
  4. Tambahkan konten ke subhalaman tersebut.
  5. Klik Publish ketika siap.

Ketika Pages Anda tercantum, Page Anak akan bersarang di bawah Halaman Induk. The Permalinks dari Pages Anda juga akan mencerminkan hirarki Halaman ini.

Dalam contoh di atas, [Permalinks [Menggunakan | Permalink]] untuk Kamerun Halaman akan: 

 http://example.com/africa/cameroon/

Mengubah URL dari Anda Pages[edit]

Untuk mengubah bagian URL (juga disebut sebagai "siput") tombol yang berisi nama Halaman Anda, gunakan "Edit" (atau "Ganti Permalinks" dalam versi WordPress yang lebih tua) dengan judul halaman pada Ubah Layar dari tertentu Page, diakses dari Pages tab WordPress Panel Administrasi.

Membuat Daftar Pages[edit]

WordPress mampu otomatis menghasilkan daftar Pages di situs Anda dalam sidebar atau footer, misalnya, menggunakan Tag Template disebut wp_list_pages () . Lihat wp_list_pages halaman untuk informasi tentang bagaimana untuk menyesuaikan bagaimana WordPress menampilkan daftar Pages di situs Anda.

Ada juga Plugin WordPress yang akan membantu Anda menampilkan daftar Pages dalam di Pos dan di daerah lain Theme WordPress Anda.

Page Template[edit]

Pages individu dapat diatur untuk menggunakan kustom spesifik 'Halaman Template' (file template Tema PHP WordPress, misalnya, saya custom-page.php-) Anda buat dalam tema Anda. Lihat Custom Page Template untuk petunjuk tentang cara membuat file template kustom untuk Halaman. Template ini Halaman baru kemudian akan menimpa default page.php Template Halaman disertakan dengan tema Anda.

Dynamic Sifat WordPress Pages[edit]

Sebuah halaman web dapat statis atau dinamis. Halaman statis, seperti halaman HTML biasa yang mungkin Anda buat dengan Dreamweaver, adalah mereka yang telah dibuat sekali dan tidak perlu dibuat ulang setiap kali orang mengunjungi halaman. Sebaliknya, halaman dinamis, seperti yang Anda buat dengan WordPress, perlu dibuat ulang setiap kali mereka dilihat; kode untuk apa untuk menghasilkan pada halaman tersebut telah ditentukan oleh penulis, namun bukan halaman itu sendiri. Ini menggunakan kode PHP yang luas yang dievaluasi setiap kali halaman dikunjungi, dan konten tersebut sehingga dihasilkan pada setiap kunjungan baru.

Hampir segala sesuatu di WordPress dihasilkan secara dinamis, termasuk 'Pages' . Semuanya diterbitkan di WordPress (Tulisan, Pages, Comments, Blogrolls, Kategori, dll ...) disimpan dalam MySQL Database. Ketika situs tersebut diakses, informasi database yang digunakan oleh WordPress Anda Template dari Anda saat ini Theme untuk menghasilkan halaman web yang diminta.

Sebuah contoh halaman statis mungkin menjadi HTML dokumen (tanpa PHP code). Masalah dengan halaman murni statis adalah bahwa mereka sulit untuk mempertahankan. Perubahan yang Anda buat untuk pengaturan WordPress Anda, Tema dan Template tidak akan disebarkan ke halaman kode hanya dalam HTML. Fitur Halaman dari WordPress dikembangkan untuk mengatasi masalah ini. Dengan menggunakan Pages, Anda tidak lagi harus memperbarui halaman statis Anda setiap kali Anda mengubah gaya situs Anda. Jika ditulis dengan benar, Pages dinamis Anda akan memperbarui bersama dengan sisa dari blog Anda.

Meskipun sifat dinamis dari Pages, banyak orang merujuk kepada mereka sebagai statis. Mereka benar-benar disebut "pseudo-statis" halaman web. Dengan kata lain, Halaman berisi informasi statis tapi dihasilkan secara dinamis . Dengan demikian, baik "statis," "dinamis," atau "pseudo-statis" dapat secara sah digunakan untuk menggambarkan sifat dari fitur WordPress Halaman.
